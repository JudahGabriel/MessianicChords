# Web Awesome Migration Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Replace Shoelace 2.20.1 with Web Awesome 3.10.0 while preserving Messianic Chords behavior, appearance, forms, and locally hosted Bootstrap icons.

**Architecture:** Keep the existing Lit component structure and migrate in five independently buildable commits. Override Web Awesome's `default` icon library once at application startup so existing icon names resolve to `/assets/icons/*.svg`; use the Shoelace-compatible Web Awesome theme as the visual baseline.

**Tech Stack:** TypeScript 7, Lit 3, Vite 8, Web Awesome 3.10.0, ASP.NET Core, ESLint 8, Spiderloop, Playwright MCP

